You are a senior full-stack mobile app product engineer building a commercial-grade productivity application called LifeOps (Life Operating System).

The goal is to build a mobile-first app that feels human-designed, calm, and trustworthy, suitable for real users and paid subscriptions.
This must not feel like a prototype or AI-generated demo.

PRODUCT VISION

LifeOps is decision-centric, not task-centric.

Instead of showing long to-do lists, the app determines the single best next action for the user based on:

Available time

Energy level

Priority and deadlines

Context (study / work / personal)

The app should actively reduce mental load and help users focus on what matters right now.

PLATFORM REQUIREMENTS

Platform: Mobile Application (Android-first, iOS-ready)

Framework: React Native or Flutter (choose what integrates best)

App must support push notifications after installation

MVP FEATURES (STRICT SCOPE)
1. Authentication

Email + password signup and login

Simple, secure, minimal UI

No social logins for MVP

2. Task Creation (Minimal & Human)

Each task includes:

Title

Category (Study / Work / Personal)

Estimated time (in minutes)

Priority (Low / Medium / High)

Optional deadline

Avoid tags, advanced settings, or clutter.

3. “Next Best Action” Engine (CORE FEATURE)

Implement a logic-based decision engine (not machine learning yet) that:

Asks user:

Available time (5 / 15 / 30 / 60+ minutes)

Current energy level (Low / Medium / High)

Selects one best-fit task

Displays only that task on the main screen

The home screen must always feel quiet and focused.

4. Focus Session

Start focus session on the selected task

Simple timer interface

Options:

Pause

Complete

Reschedule

Use calm, respectful language (no gamification)

5. Push Notifications (IMPORTANT)

Implement mobile notifications for:

Focus session reminders

Gentle nudges when user drifts from a scheduled task

Daily reminder to check “Next Best Action”

Optional end-of-day reminder to view Daily Replay

Notification tone must be:

Polite

Supportive

Non-guilt-inducing

Example:

“You have a clear 20 minutes right now. Ready to continue?”

6. Daily Replay (Auto Summary)

At the end of each day:

Show completed tasks

Time spent

Tasks postponed or missed

A short AI-generated reflection (1–2 sentences, calm and neutral)

Example tone:

“Short focus sessions worked well today. Planning similar blocks tomorrow may help.”

User interaction should be optional.

UI / UX REQUIREMENTS (VERY IMPORTANT)
Design Principles

Professional

Calm

Minimal

Human-crafted

Color Palette (STRICT)

Background: off-white or light gray

Primary text: charcoal / dark gray

Accent color: muted blue or muted green

Buttons: solid, neutral

No purple

No pink

No neon

No gradients

Typography

System fonts or Inter

Clear hierarchy

Generous spacing

Comfortable line heights

UX RULES

No fancy animations

No productivity jargon

No emojis

No cluttered dashboards

No “AI hype” language

This app should feel like something a serious professional or student would trust and pay for.

TECH STACK

Frontend: React Native or Flutter

Backend: Node.js or Java (Spring Boot), choose cleanest integration

Database: Relational (PostgreSQL or similar)

Notifications: Firebase Cloud Messaging or equivalent

Architecture must be scalable and clean

CODE QUALITY EXPECTATIONS

Readable, well-structured code

Clear naming conventions

Logical separation of concerns

Comments where reasoning matters

Avoid overengineering

Code should look human-written, not auto-generated.

DELIVERABLES

Fully working mobile MVP

Clean, production-ready UI

Logical Next Best Action engine

Push notification support

Scalable foundation for future AI features

DEVELOPMENT MINDSET

Think like a product engineer, not a hackathon participant

Prioritize clarity and usability

Prefer simplicity over cleverness

Build something users would pay for monthly
